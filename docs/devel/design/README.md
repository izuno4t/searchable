# docs/devel/design/

現在有効な設計内容を配置する。

採用理由・代替案・判断影響は記録しない（それらは [`../adr/`](../adr/) に記録する）。

## 構成

| パス | 内容 |
| --- | --- |
| [`architecture/`](architecture/) | システム全体構造、責務分割、依存関係、主要コンポーネント |
| [`application/`](application/) | アプリケーション内部設計、レイヤー、ユースケース、サービス構成 |

`database/`・`interface/`・`batch/` は現状未使用（必要になった時点で追加する）。

## 関連 ADR

設計判断の履歴は [`../adr/`](../adr/) を参照。
