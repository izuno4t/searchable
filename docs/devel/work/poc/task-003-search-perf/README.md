# PoC: Lucene 検索性能テスト（10万件）

TASK-003 用の性能検証。10万件の合成日本語文書を Lucene + Kuromoji で
インデックス化し、検索レスポンスが目標値（500ms以内）を満たすことを確認する。

## 実行

```bash
mvn -q compile exec:java
```

## シナリオ

1. 合成日本語コーパス10万件を生成し `MMapDirectory` にインデックス化
2. ウォームアップクエリ 100 回
3. 計測クエリ 1,000 回（14種のクエリをローテーション）
4. 最小・平均・p50・p95・p99・最大レイテンシを出力
5. 500ms超え件数を集計

## 実測結果

| 項目 | 値 |
| --- | --- |
| インデックス作成（10万件） | 6,020 ms |
| 平均レイテンシ | 0.03 ms |
| p50 | 0 ms |
| p95 | 0 ms |
| p99 | 1 ms |
| 最大 | 1 ms |
| 500ms超え | 0 / 1,000 |

> 計測環境: macOS / Apple Silicon, Java 21, Lucene 10.2.0,
> `MMapDirectory`, `JapaneseAnalyzer`

## 結論

- 検索レイテンシは目標値500msに対して **3桁以上のマージン**
- 10万件規模ではボトルネックにならない
- 性能要件（要求仕様3.1.1）は十分に達成可能

## 注意点

- 本PoCは単純な合成文書による測定
- 実運用ではフィールド数、ハイライト処理、フィルタ等で
  レイテンシが増える可能性があるが、桁単位のマージンがあるため
  Phase 1 で 500ms 以内の達成は現実的
