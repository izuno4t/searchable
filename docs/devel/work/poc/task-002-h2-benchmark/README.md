# PoC: H2 Database メタデータ操作ベンチマーク

TASK-002 用ベンチマーク。H2 Database の組み込みモード（MVStore）で
10万件規模のメタデータ操作性能を計測する。

## 実行

```bash
mvn -q compile exec:java
```

## シナリオ

1. テーブル `namespaces` を作成（PK + 設定情報 + ドキュメント数）
2. 10万件をバッチINSERT（バッチサイズ 1,000）
3. `SELECT COUNT(*)` で全件カウント
4. PK指定SELECTを1,000回
5. PK指定UPDATEを1,000回

## 実測結果

| 操作 | 件数 | 合計時間 | 1件あたり |
| --- | --- | --- | --- |
| INSERT（バッチ1000） | 100,000 | 456 ms | 0.005 ms |
| SELECT COUNT(*) | 100,000 | 7 ms | - |
| SELECT BY PK | 1,000 | 25 ms | 0.025 ms |
| UPDATE BY PK | 1,000 | 28 ms | 0.028 ms |

> 計測環境: macOS / Apple Silicon, Java 21, H2 2.3.232（組み込みファイル）

## 観察された挙動

- バッチINSERTで10万件を1秒未満で投入できる
- PKアクセスは1件あたり数十マイクロ秒、要件500ms以内に対して
  メタデータDBがボトルネックになる懸念はない
- MVStoreによる組み込みファイル運用で十分な性能
- リソース確保（DBファイル作成）も含めて初期化が高速

## 結論

H2 Database 2.3.x（組み込みモード）は本プロジェクトの
メタデータDB要件を十分に満たす。
