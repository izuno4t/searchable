# Searchable - 要求仕様書

## 1. プロジェクト概要

### 1.1 目的
軽量で高速な全文検索・ベクトル検索機能を提供するJavaライブラリ。**日本語を主要対象言語**とし、Webアプリケーションへの組み込み、REST API経由での利用、MCPツールとしての利用など、多様なユースケースに対応する。

### 1.2 主要な特徴
- **日本語最適化**: 日本語形態素解析と日本語特化の埋め込みモデル
- **ハイブリッド検索**: 全文検索とベクトル検索を単独または組み合わせて利用可能
- **マルチテナント対応**: Namespaceによる論理的な分離
- **マルチAPI**: Java API、REST API、MCPサーバーとして提供
- **高速レスポンス**: インメモリアーキテクチャによる高速検索
- **プラグイン拡張**: データソースや検索処理のカスタマイズ

### 1.3 主なユースケース
- Webアプリケーションへの簡易検索機能の組み込み
- MCPツール用のドキュメント検索サーバー
- 複数のドキュメントソースの統合検索管理

## 2. 機能要件

### 2.1 検索機能

#### 2.1.1 全文検索
- キーワードベースの高速全文検索
- **日本語形態素解析**（Kuromoji/Sudachiによる分かち書き）
- **日本語の適切なトークン化**（助詞、助動詞の処理）
- マルチバイト文字対応（UTF-8）
- フィールド指定検索
- ファジー検索、ワイルドカード検索
- ハイライト機能（日本語テキスト対応）

#### 2.1.2 ベクトル検索（MVP範囲）
- Javaのみで完結する埋め込み処理（Onnx Runtime使用）
- **日本語対応埋め込みモデル**（multilingual-e5-small/intfloat等）
- セマンティック類似度検索
- レスポンス性能: 500ms以内（単一Namespace）

#### 2.1.3 ハイブリッド検索
- 全文検索とベクトル検索の組み合わせ
- 検索順序の設定
  - シーケンシャル: 全文検索 → ベクトル検索 または逆順
  - パラレル: 両方同時実行してマージ
- AI統合（オプション）: 複数の検索結果をAIで要約・統合

#### 2.1.4 検索結果
- ページネーション対応
- スコアリング
- メタデータ付与（ドキュメントソース、更新日時等）

### 2.2 Namespace機能

#### 2.2.1 基本機能
- 論理的なインデックス分離
- 動的な作成・削除（API経由）
- Namespaceごとの検索実行
- 複数Namespace横断検索

#### 2.2.2 設定管理
- グローバルデフォルト設定
- Namespace個別設定（個別設定が優先）
- 設定可能項目:
  - 検索アーキテクチャ（全文検索/ベクトル検索/ハイブリッド）
  - 検索順序
  - AI統合の有効/無効
  - 埋め込みモデルの選択
  - その他検索パラメータ

#### 2.2.3 権限管理（設計のみ、実装は将来）
- ユーザー/ロール管理を見据えた設計
- MVPでは権限チェックなし（組み込み側で制御）

### 2.3 インデックス管理

#### 2.3.1 ドキュメント登録
対応フォーマット:
- **MVP範囲**:
  - テキスト系: Plain Text, Markdown, AsciiDoc
  - PDF
  - HTML
- **将来対応**:
  - Office系: Word, Excel, PowerPoint
  - Google Docs, Apple Pages等（PDF変換での対応を推奨）

#### 2.3.2 インデックス更新
- ドキュメント変更の検知
- 非同期インデックス更新
- 差分更新対応
- バッチ更新API

#### 2.3.3 永続化
- ファイルシステム保存
- DB保存
- 両方選択可能（設定により切り替え）

#### 2.3.4 バックアップ・リストア
- 自動バックアップ機能
- 手動エクスポート/インポートAPI
- バックアップスケジュール設定

### 2.4 API提供

#### 2.4.1 Java API
- ライブラリJARとして配布
- 主要機能:
  - 検索実行
  - インデックス管理
  - Namespace管理
  - 設定管理

#### 2.4.2 REST API
- 認証方式: API Key、Bearer Token
- エンドポイント:
  - `/search`: 検索実行
  - `/namespaces`: Namespace管理
  - `/index`: インデックス管理
  - `/admin`: 管理機能
- OpenAPI仕様書の提供

#### 2.4.3 MCP（Model Context Protocol）サーバー
- 主要機能:
  - ドキュメント検索
- 将来拡張:
  - インデックス更新

### 2.5 UI機能

#### 2.5.1 管理UI（Thymeleaf、優先実装）
- Namespace管理
  - 一覧表示、作成、編集、削除
  - 設定変更
- インデックス管理
  - インデックス状態確認
  - 手動インデックス更新
  - ドキュメント登録/削除
- システム設定
  - グローバル設定
  - バックアップ設定
- モニタリング
  - インデックスサイズ
  - 検索パフォーマンス統計

#### 2.5.2 検索UI（React、サンプル実装）
- シンプルな検索インターフェース
- 検索結果表示
- ページネーション
- ハイライト表示
- ライブラリ利用のリファレンス実装として提供

### 2.6 プラグイン機構

#### 2.6.1 MVP範囲
- **データソースプラグイン**
  - カスタムデータソースからのドキュメント取得
  - プラグインインターフェースの定義
  - JARファイル配置による読み込み

#### 2.6.2 将来拡張
- カスタム検索フィルタ
- 文書パーサープラグイン
- カスタムスコアリング

### 2.7 AI統合（オプション機能）

#### 2.7.1 対応AI API
- OpenAI API
- Anthropic Claude API
- ローカルLLM（Ollama等）
- 複数AI APIの切り替え対応

#### 2.7.2 AI機能
- 検索結果の要約・統合
- セマンティック検索の拡張
- 設定によるON/OFF切り替え

#### 2.7.3 制御
- タイムアウト設定（調査中）
- 費用制限はAIサービス側で管理

## 3. 非機能要件

### 3.1 性能要件

#### 3.1.1 検索レスポンス
- **目標**: 500ms以内（単一Namespace検索時）
- **想定文書数**: Namespaceあたり最大100,000件
- **同時接続数**: 実行環境依存（アプリケーション側での制約なし）

#### 3.1.2 インデックスサイズ
- メモリベース: 数GB程度まで対応
- ディスクベース: 実行環境依存で制限なし

#### 3.1.3 スケーラビリティ
- Namespace単位での分散可能な設計
- 水平スケーリングは実行環境に依存

### 3.2 可用性・信頼性
- インデックスの永続化による再起動後の復旧
- 自動バックアップ機能
- エラーハンドリングとログ出力

### 3.3 保守性
- SLF4J + Logbackによる構造化ログ
- 設定ファイルによる柔軟な構成管理
- プラグイン機構による拡張性

### 3.4 セキュリティ

#### 3.4.1 認証・認可（MVP範囲外、設計考慮）
- API Key/Bearer Token認証（REST API）
- 将来的なユーザー/ロール管理を見据えた設計
- MVPでは組み込み側で制御

#### 3.4.2 データ保護
- インデックスデータの暗号化（将来検討）
- 通信の暗号化（HTTPS、組み込み側で対応）

## 4. 制約事項・前提条件

### 4.1 実行環境
- Java 21以上
- メモリ: Namespaceあたり数GB推奨（インメモリモード時）
- ディスク: インデックスサイズに応じた容量

### 4.2 ライセンス
- 使用するOSSライブラリのライセンス確認
- ライブラリ配布時のライセンス明記

### 4.3 サポート範囲
- 日本語を含むマルチバイト文字対応
- UTF-8エンコーディング

## 5. 用語集

- **Namespace**: 論理的なインデックス分離単位
- **埋め込み（Embedding）**: テキストをベクトル表現に変換すること
- **ハイブリッド検索**: 全文検索とベクトル検索の組み合わせ
- **MCP**: Model Context Protocol、AIツールとの連携プロトコル
