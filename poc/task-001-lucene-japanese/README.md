# PoC: Lucene + Kuromoji 日本語全文検索

TASK-001 用の簡易PoC。Apache Lucene と Kuromoji
（`JapaneseAnalyzer`）で日本語の全文検索が動作することを確認する。

## 実行

```bash
mvn -q compile exec:java
```

## 検証項目

- 日本語サンプル文書（5件）のインメモリ・インデックス化
- 5種類のクエリ実行
- BM25によるスコアリング
- ハイライト機能（`<mark>` タグ）

## 実測結果

| クエリ | ヒット件数 | 実行時間 |
| --- | --- | --- |
| 日本語 | 0 | 4 ms |
| 形態素解析 | 3 | 5 ms |
| Lucene 検索 | 2 | 0 ms |
| ハイブリッド | 1 | 0 ms |
| Kuromoji 辞書 | 3 | 0 ms |

## 観察された挙動

- 形態素分割により「Kuromoji」「形態素」「ハイブリッド」等が独立トークン
  として正しく検索される
- 英語と日本語の混在クエリ（例: 「Lucene 検索」）も正しく動作
- ハイライト機能が `<mark>` タグで動作
- BM25によるスコアリングが機能（同義語的な文書が高スコア）
- 短い名詞（例: 「日本語」）が POS フィルタにより検索対象から除外される
  ケースがあるため、サービス層では `JapaneseAnalyzer` の POS 設定または
  カスタム Analyzer の検討が必要（Phase 1 ランキングチューニング要件で対応）

## 結論

Lucene 10.2.0 + Kuromoji（`JapaneseAnalyzer`）は日本語全文検索の
基本要件を満たす。Phase 1 ではこの構成を採用し、POS フィルタの調整は
要件発生時にチューニングする。
